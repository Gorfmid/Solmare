# BOOK ONE RUNTIME LOOP — AI GENERATION ENGINE

## PURPOSE
This defines how Book One is generated end-to-end.

---

## STEP 1 — SELECT CHAPTER INPUTS

From chapter skeleton:
- pick chapter objective
- assign POV
- assign system/location
- assign ship state

---

## STEP 2 — VALIDATE AGAINST RULES

Check:
- continuity_rules.md
- ship constraints
- empire doctrine
- anomaly state rules

If invalid → revise inputs before generation.

---

## STEP 3 — GENERATE INCIDENT CHAIN

Create 3–7 escalating events:
- must increase tension
- must not introduce new rule-breaking systems
- must lead toward discovery

---

## STEP 4 — GENERATE DISCOVERY

Select ONE:
- system truth shift
- anomaly behavior change
- contradiction in data

---

## STEP 5 — APPLY STATUS SHIFT

Modify at least one:
- ship condition
- crew psychology
- command trust level
- mission clarity

---

## STEP 6 — OUTPUT CHAPTER

Render using output_format_contract.md exactly.

---

## STEP 7 — CONTINUITY LOCK

Append chapter state to runtime log:
- ship state
- known anomalies
- political awareness level
- unresolved contradictions

This ensures persistence across chapters.
