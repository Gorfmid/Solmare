# Character Pass

**Router:** `Writing_System/revision_pass.md`  
**Lifecycle:** Stage 2 — runs **after** structural revision (Passes 1–4), before Final Candidate / Writing Pass 1  
**Trigger phrases:**  
- *Character Pass on Chapter X*  
- *Apply character pass to Chapter X*  
- *Run character pass*

**Load with:** Relevant `Characters/` sheets for every named person on the page. Do not invent personality that contradicts a sheet.

---

## Purpose

This pass occurs **after** the structural rewrite.

Do **not** change the plot.  
Do **not** add or remove scenes.  
Do **not** alter pacing unless required for character behavior.

Instead, improve the chapter by making every character behave consistently with their established character sheet.

**People before process** still governs (`Style_Guide/people_before_process.md`). This pass is how you verify that the people on the page are *these* people — not interchangeable professionals.

---

## For every named character present

### 1. Verify Character Voice

Does this dialogue sound like this specific character?

Could another character say these same lines?

If yes, rewrite until the dialogue is unmistakably theirs.

Use the sheet’s **Authentic voice** / **Never sounds like** notes. Blind-read test: remove the name tag — can you still tell who is speaking?

---

### 2. Verify Character Perspective

Each character notices different things.

Ensure every observation matches their expertise and personality.

Examples (*Kestrel Veil* Book Two+):

| Character | Tends to notice |
|-----------|-----------------|
| **Calder** | Leadership, uncertainty, people, decisions that land on his desk |
| **Thessaly** | Workload, readiness, crew health, what will break next |
| **Park** | Patterns, relationships, communication, behavior, who stopped joking |
| **Reyes** | Movement, spatial relationships, approach geometry |
| **Tanaka** | Details, checklists, subtle mistakes in sequence |
| **Brenner** | Engineering problems, what the hull can actually do |
| **Hale** | Threats, emitters, rooms before he enters them |
| **Okwelu** | Physical and mental health, rest, who is holding |
| **Mehta** | Environmental systems, filters, atmosphere |
| **Solis** | Security, access, corridors, who has authority where |
| **Ortega** | Scientific anomalies, beautiful wrong numbers |
| **Vex** | Pattern library matches, long-watch geometry, what Dominion already knows |
| **Orlen** | Lexicon precision, what is finished vs only close |
| **Vorin** | What is confirmed vs left open; dual-custody honesty |

No two characters should interpret the same event in exactly the same way.

---

### 3. Verify Character Behavior

Characters should solve problems the way their character sheet describes.

Avoid generic competence.

Show each person’s unique approach.

---

### 4. Reduce Interchangeability

If dialogue or observations could be reassigned to another character without changing the scene — rewrite them.

Every contribution should feel unique.

---

### 5. Use Quiet Character Moments

Look for opportunities to reinforce personality through small actions:

- Routine  
- Habits  
- Body language  
- Objects  
- Small jokes  
- Professional rituals  

Avoid adding exposition.

---

### 6. Strengthen Relationships

Where appropriate, reinforce established relationships through natural interaction.

Use restraint.

Professionals rarely explain how they feel about each other.

Readers should infer relationships through behavior.

See also: `competence_before_attraction.md`

---

### 7. Preserve Plot

This pass is **not** a structural rewrite.

- Keep scene order  
- Keep chapter length approximately the same  
- Do not invent new subplots  
- Only improve characterization  

---

## How to run

1. List every named character in the chapter.  
2. Open each character sheet.  
3. Scene by scene: voice → perspective → behavior → interchangeability → quiet moments → relationships.  
4. Surgical edits only. If a line needs a new scene to fix voice, you are in the wrong pass — flag it; do not rebuild structure here.  
5. Report briefly (below).

---

## Exit check

- [ ] Every named character’s dialogue passes the blind-read / reassignment test  
- [ ] Observations match expertise (no two people reading the event identically)  
- [ ] Behavior matches sheet method, not generic competence  
- [ ] Quiet habits present where natural (not forced)  
- [ ] Relationships shown through behavior, not explained  
- [ ] Plot, scene order, and approximate length unchanged  

---

## Success criteria

When finished, readers should feel they know the characters better **without noticing that the plot changed**.

Every scene should feel as though **only these specific people** could have lived it.

---

## Report (short)

1. Characters checked (list)  
2. Voice / interchangeability fixes (who, what)  
3. Perspective / behavior fixes (who, what)  
4. Quiet moments / relationship beats strengthened  
5. Left unchanged because already on-voice
