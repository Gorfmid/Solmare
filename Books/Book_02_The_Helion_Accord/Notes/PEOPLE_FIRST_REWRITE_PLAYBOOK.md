## What the rewrite was

A people-first rewrite is **not** line polish. It rebuilds a chapter that is procedurally correct but emotionally dead.

### Goal

Make the chapter feel like characters choosing under pressure — not a protocol manual with names attached.

### Method (do this on future chapters that feel “correct but dead”)

1. **State the chapter job in one sentence** as a human outcome (who changes, what they decide), not as a policy deliverable.
2. **Cut or demote** scenes whose only job is to re-teach spent facts, re-list roster, or re-justify wording already locked.
3. **Put people in the gaps:** corridor breaks, private agreements, competence noticing competence, someone arriving from real work with a practical fix.
4. **One memorable concrete example** for any institutional idea — then move. Do not prove the point five ways.
5. **Plant life; pay off loss later.** If something will be destroyed next chapter, this chapter shows it as ordinary and wanted — not foreshadowed doom.
6. **End on the next useful action**, not a narrator caption about what the moment meant.
7. **Plain speech pass:** if the author asks *what does this mean?*, rewrite immediately. No soft nouns, no cute synonym rotation, no institutional gift language.

### Working test

Could you remove every policy line and still feel who these people are and what they chose?  
If yes → policy is doing too much; character is doing too little. Rewrite.

## Common changes (apply to future chapters)

### A. Structure

| Change | Do this |
|--------|---------|
| People first | Lead with choice, voice, relationship temperature. Policy serves that. |
| One scene, one purpose | After the point lands, leave. No second conversation that restates the goal. |
| Corridor / lived texture | Between protocol blocks, let people eat, drink, notice, disagree as humans. |
| Practical solver beat | Someone (often engineering/security) arrives from real work with a fix politics hasn’t caught up to. |
| Private channel stays private | Internal Council/SSD notes are not briefed to the other empire by default. |
| Side-channel realism | Relay officers know Segment traffic; they do not magically know secret visit plans. |

### B. Character

| Change | Do this |
|--------|---------|
| Competence before personality | Respect is earned by prep, discipline, useful work — never by appearance. |
| Need ≠ affection | Working agreements can be blunt (“we have to”) without friendship speeches. |
| Ask, don’t rescue | Mentors ask the question that lets command decide. |
| Career progression | Characters who leave a billet **advance** (transfer/promotion/project). Never vanish. |
| Protected cast need a voice | Recurring names must occasionally do meaningful work on the page. |

### C. Language (common cuts / swaps)

| Cut / avoid | Prefer |
|-------------|--------|
| Soft-noun / *label* / invent-words meta | Concrete failure or ask |
| Institutional *gift* for maps/disclosure | Name the real offer or refuse |
| *Polite fiction* / *polite X* | Name the real ask or lie |
| *Shared strip* | *Shared lexicon* (when that is meant) |
| Default *fringe* | Outer corridor / border / Neutral Zone edge |
| *Pressure* / *contour* as threat stand-ins | *Peligar advance* / *systems going quiet* / *missing returns* |
| *Depth maps* | *Strategic charts* / *strategic archive* |
| Staff poetry (*Shared channel only*, *freeze the venue*) | Ordinary custody rules in full sentences |
| Sentence-starting *And* piles; “prettier reason to say no” | One plain clause naming the refusal |
| Narrator doom wink / theme caption after the beat | Next action or logged decision |
| Briefing stacks / five examples | One memorable example |

### D. Foreshadowing / plants

| Change | Do this |
|--------|---------|
| Quiet plant | Show ordinary life + a practical future use (visit, survey, school petition). |
| No doom caption | Do not tell the reader the place is already dead. |
| Payoff chapter owns the horror | Loss lands harder if the prior chapter wanted the place to stay useful. |

### E. Continuity hygiene while rewriting

| Change | Do this |
|--------|---------|
| Do not re-teach | Use spent facts; spend the page on today’s fight. |
| Seed vs approval | A named proposal + survey is not the same as full institutional greenlight + roster. Stage them. |
| One person, one name | Fix identity drift immediately. |
| Name collision lock | Different characters may not share a surname on the page without clear rank/role tags. |

## How this fits the Revision Pass

When the author says **Revision Pass on Chapter N**, the agent loads the router (`Writing_System/revision_pass.md`) then runs:

1. **Pass 1 — People** ← this playbook’s home (`01_people.md`)  
2. **Pass 2 — Story / structure**  
3. **Pass 3 — Clarity / language**  
4. **Pass 4 — Continuity + preflight**  

All four still complete. Load size stays small because only one pass file is open at a time.
